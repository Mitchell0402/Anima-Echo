# Visual Renovation Plan

Status: Implementing - first-pass vector asset batch generated
Last updated: 2026-06-26

## Goal

Rebuild Anima Echo's remaining non-animated visual layer into one
coherent cozy top-down pixel-art direction, using committed vector
source files as the canonical assets. The target is a warm, handmade,
farming-RPG-adjacent feel similar in mood to Stardew Valley, without
copying its exact UI, sprites, map shapes, icons, or palette.

This plan started as documentation only. The 2026-06-26 implementation
pass has now generated the first-pass vector batch, but it still does
not edit scenes, wire UI code, or mark the generated art final.

## Current State Summary

The current project has three visual strata:

- **Keep as legacy animation references for now:** player animation
  sheets and gnoll enemy animation sheets. They are outside this static
  vector batch and still need sidecar metadata later.
- **Generated first-pass vector placeholders:** the previous 23
  imagegen PNGs, older static world art, mine props, the minecart, and
  the town map now have sibling SVG sources, PNG runtime exports, and
  sidecars generated by `scripts/tools/generate_vector_assets.py`.
- **Redesign and skin next:** UI surfaces are mostly Godot `Panel`,
  `PanelContainer`, `ColorRect`, `Label`, and text `Button` nodes. They
  work functionally. The UI skin texture batch exists, but the runtime
  UI still needs a themed skin, icon usage, and consistent popup/dialog
  treatment.

Important current constraints:

- Town world size is fixed at `1152x648`.
- Town UI already follows the world/camera/UI layer model.
- Mine scene still uses older scene layout and hard-coded TileMap data.
- The generated P0/P1/P2 assets now have editable SVG sources and
  regenerated runtime PNG exports, but remain `placeholder` until human
  art review.
- `scenes/mine/small_mine.tscn` currently references a broken path:
  `res://assets/gj/environment/stones_1.png`.

## Art Direction

Use a compact, readable top-down pixel-art style:

- **Mood:** warm, friendly, grounded mining-town fantasy.
- **World palette:** earthy greens, warm browns, mossy stone, muted gold,
  mineral glow accents, and restrained blue/cyan for oxygen or magic.
- **UI palette:** parchment cream, honey wood, dark walnut outlines,
  brass/gold highlights, warm grey shadows, readable dark text.
- **Shape language:** chunky readable silhouettes, beveled pixel edges,
  rounded wood panels, soft carved corners, small decorative screws or
  knots, and simple stone/mineral motifs.
- **Perspective:** top-down or three-quarter top-down. No side-view props
  unless explicitly used as an icon.
- **Line weight:** consistent 1-2 px dark outline at final resolution.
- **Texture:** pixel-clean, handmade, not painterly or blurred.
- **Shadows:** soft 2 px contact shadow for world sprites. UI icons do not
  need world shadows.
- **Transparency:** individual sprites and UI pieces should have fully
  transparent backgrounds unless they are full-screen/full-panel textures.
- **Source format:** author all newly generated or renovated static art
  as SVG/vector. The vector should use clean filled shapes, hard edges,
  limited palette regions, and pixel-aligned exports so the runtime PNG
  still reads like pixel art.
- **Runtime export:** keep PNG exports beside SVG sources when Godot
  scenes/scripts need PNG textures. The PNG is not the source of truth.

Avoid:

- Direct Stardew Valley copies.
- Large semi-realistic rendered objects.
- Mixed high-resolution illustration and pixel-art in the same screen.
- Text baked into art, except intentional symbol-like UI marks.
- Reusing mine world pickup art as warehouse UI icons.
- Treating imagegen raster output as the final editable asset.

## Vector-First Asset Policy

All future generated assets in this renovation should have:

- A canonical vector source, preferably `assets/.../<name>.svg`.
- A Godot runtime export, usually `assets/.../<name>.png`.
- A PNG sidecar, `assets/.../<name>.png.meta.md`, with:
  `vector-source`, `runtime-export`, `source`, `license`, dimensions,
  palette, status, and regeneration notes.

If an image model is used, its bitmap output is only a reference layer or
concept. The final committed asset must be re-authored or converted into
clean vector art before it can move to `implemented`.

## Risks

- Replacing atlas textures in place may break TileMap cell assumptions if
  the generated art changes the atlas structure.
- A full town map redraw can invalidate NPC placement and walkable space
  even if the file path stays the same.
- UI art can reduce readability if text backgrounds, font size, and button
  height are not tested with Chinese strings.
- A one-shot generation of all 86 static assets could produce inconsistent
  palettes; batching with sample approval is safer.
- Directly imitating Stardew Valley can create an IP/style-copy problem;
  prompts should describe a warm cozy pixel RPG direction instead of asking
  for a clone.
