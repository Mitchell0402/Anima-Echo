# 矿洞地牢模式重设计

Status: 设计确认 / 待开发
Last updated: 2026-06-30

> ✅ 以下关键决策已确认：
> - 玩家攻击：J 键近战攻击
> - 怪物掉落：只掉铜板小奖励，主要产出靠矿山节点
> - Boss 房间：不锁门，打不过可以逃
> - 房间大小：640×360 战斗区域

> 将现有单一手绘矿洞场景替换为以撒的结合（Binding of Isaac）风格的地牢生成系统。
> 轻量化：单层 4-6 房间 + 1 Boss。不做肉鸽，做局外成长。
> 5 档难度（Lv1-5），不同怪物池和掉落。

---

## 一、参考：以撒的结合地牢生成算法

以撒的算法分两阶段：**① 生成房间布局图 → ② 填充每个房间的内容**。

### 1.1 布局生成（Floorplan）

```
基于 9x8 的网格，起始房间在网格中央(35号)
使用 BFS 向外扩展：
  - 对队列中每个房间，尝试向 4 个方向扩展
  - 邻居已有房间 → 跳过
  - 邻居已有 ≥2 个邻接房间 → 跳过（禁止环路）
  - 房间数量已够 → 跳过
  - 50% 随机 → 跳过
  - 否则：标记邻居为有效房间，加入队列
```

**关键规则**：
- **无环路**（秘密房间除外）——走廊永远是树枝状
- 最远的死胡同 = Boss 房间
- 特殊房间按「离起始房间最远→最近」的顺序依次放置在死胡同中
- 相邻房间必然有门（在墙壁正中央），且所有房间都可从四方向进入

### 1.2 我们的简化版

| 以撒原版 | 我们的简化版 |
|----------|-------------|
| 每层 7-20 个房间 | 每层 4-6 房间 + 1 Boss |
| 多个特殊房间（商店、宝箱、牺牲…） | 只有 3 种房间：安全/战斗/Boss |
| 9x8 网格 | 5x5 或动态网格（足够小） |
| 房间内容从大量预制池中随机选取 | 房间内容按难度动态生成（怪物组合） |
| 多层级（每章 2 层） | 单层（一次矿洞探险） |
| 全方向射击+飞行怪物 | 四方向移动+近战/直线弹幕 |

## 二、地图设计

### 2.1 房间类型

| 房间类型 | 数量 | 说明 |
|---------|------|------|
| **起始房间** (Safe Room) | 1 | 玩家出生点。无敌人。有矿车出口（返回城镇）。 |
| **战斗房间** (Combat Room) | 4-6 | 门会锁住，必须杀光敌人才能解锁。清除后中央出现 1-3 个矿山节点。 |
| **Boss 房间** (Boss Room) | 1 | 位于最远的死胡同。击败 Boss 后出现高级矿山节点 + 奖励掉落。 |

### 2.2 房间数量公式

```
rooms_per_level = random(4, 6)   // 4-6 个战斗房间
total_rooms = 1(起始) + rooms_per_level(战斗) + 1(Boss)  // 总计 6-8 个房间
```

不同难度级别可微调：
| 难度 | 战斗房间数 | 总房间数 |
|------|-----------|---------|
| Lv1 | 4 | 6 |
| Lv2 | 4-5 | 6-7 |
| Lv3 | 5 | 7 |
| Lv4 | 5-6 | 7-8 |
| Lv5 | 6 | 8 |

### 2.3 布局生成算法（GDScript 伪代码）

```gdscript
# MineDungeonGenerator
# 在 7x5 网格上生成（足够容纳 6 个房间 + 走廊空间）

const GRID_WIDTH = 7
const GRID_HEIGHT = 5
const START_CELL = Vector2i(3, 2)  # 中央偏左

func generate_floorplan(target_rooms: int) -> Dictionary:
    var occupied = {}  # Vector2i -> room_type
    var queue = [START_CELL]
    occupied[START_CELL] = "start"
    
    var room_count = 1
    var dead_ends = []
    
    while queue.size() > 0:
        var cell = queue.pop_front()
        var added_any = false
        
        # 随机打乱四个方向
        var dirs = [Vector2i.UP, Vector2i.DOWN, Vector2i.LEFT, Vector2i.RIGHT]
        dirs.shuffle()
        
        for dir in dirs:
            var neighbor = cell + dir
            
            # 边界检查
            if not _in_bounds(neighbor):
                continue
            
            # 邻居已有房间 → 跳过
            if occupied.has(neighbor):
                continue
            
            # 邻居已有 ≥2 个邻接房间 → 跳过（禁止环路）
            if _count_occupied_neighbors(neighbor, occupied) >= 2:
                continue
            
            # 房间数量已够 → 跳过
            if room_count >= target_rooms:
                continue
            
            # 70% 概率扩展（比原版 50% 略高，因为房间少）
            if randf() < 0.3:
                continue
            
            # 放置房间
            occupied[neighbor] = "combat"
            queue.append(neighbor)
            room_count += 1
            added_any = true
        
        # 如果这个房间没有扩展出任何邻居 → 它是死胡同
        if not added_any:
            dead_ends.append(cell)
    
    # 验证：实际房间数是否达到目标？
    # 死胡同数量是否 ≥ 1（至少需要 1 个放 Boss）？
    # Boss 不能紧邻起始房间
    if room_count < target_rooms or dead_ends.is_empty():
        return generate_floorplan(target_rooms)  # 重试
    
    # 选最远的死胡同放 Boss 房间
    var boss_cell = _pick_farthest_dead_end(dead_ends, START_CELL)
    occupied[boss_cell] = "boss"
    
    return occupied
```

### 2.4 房间场景设计

每个房间是一个 640×360 px 的方形区域。通过 Camera2D 以 2× 整数缩放填满 1280×720 视口，一次只显示当前房间（类似以撒的结合）。

**房间结构（每个房间内部）**：
```
┌─────────────────────┐
│  ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓  │  ← 上墙
│  ▓               ▓  │
│  ▓   战斗区域    ▓  │  ← 640×360 内部区域
│  ▓               ▓  │     怪物在这里巡逻
│  ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓  │
│     [门]  [门]       │  ← 根据邻接房间开门
└─────────────────────┘
```

**相机系统**：
- 2× 整数缩放（640×2=1280, 360×2=720），像素完美
- 玩家穿过门时相机瞬间切换到新房间中心
- 0.3 秒穿门冷却防止反复触发

**门机制**：
- 房间之间的门在墙壁正中央
- 当玩家进入战斗房间时：所有门关闭（变成石墙/铁栏）
- 杀光房间内所有敌人后：门打开，矿山节点出现
- Boss 房间：门不会关闭（或者关闭但击败 Boss 后打开+出现矿山）
- 已清除的房间可以自由来回

### 2.5 起始房间

- 有一个固定的矿车出口（`MinecartExit`）→ `GameRuntime.end_mine_run()`
- 有一个氧气补给泵（`OxygenPump`）
- 可能有 1-2 个宝石散落在地上作为「欢迎」
- 所有门从里面不会被锁（可以随时进去冒险，但出来不锁）

## 三、怪物设计（5 种小怪）

所有怪物参考以撒的结合设计，适配 2D 四方向移动和近战/直线弹幕。

### 3.1 怪物总览

| # | 名称 | 英文参考 | 行为模式 | 速度 | HP | 伤害 | 出现难度 |
|---|------|---------|---------|------|-----|------|---------|
| 1 | **矿蝇** | Fly | 围绕玩家转圈，周期性冲刺 | 120/转圈, 250/冲刺 | 1 | 10 | Lv1+ |
| 2 | **碎石怪** | Gaper | 缓慢走向玩家，每 3 秒加速冲撞 | 80/慢, 300/冲撞 | 2 | 15 | Lv1+ |
| 3 | **爆裂晶** | Mulliboom | 追踪玩家，近身 1 秒后自爆 | 200 | 1 | 20(AOE) | Lv2+ |
| 4 | **水晶蛛** | Trite | 跳跃移动，落地时发射 4 方向弹幕 | 跳跃180, 弹幕300 | 2 | 12(弹幕) | Lv3+ |
| 5 | **幽魂矿工** | Bony | 保持距离，发射直线弹幕 | 100, 弹幕250 | 3 | 18(弹幕) | Lv4+ |

### 3.2 详细设计

#### 1. 矿蝇（Mine Fly）
- **外观**：深灰色飞虫，翅膀带淡蓝色矿石光泽
- **AI**：围绕玩家 120-180 px 距离转圈（顺时针/逆时针随机），每次转 2-3 圈后随机向玩家冲刺
- **冲刺**：直线冲过 250 px，碰到墙壁反弹
- **碰撞伤害**：10 HP，可被攻击击退
- **特色**：数量多时形成弹幕网，是最基本的小怪

#### 2. 碎石怪（Rubble Gaper）
- **外观**：人形石堆，头部裂开形成大口
- **AI**：缓慢走向玩家，每 3-4 秒发出一声咆哮后加速冲撞 1 秒
- **冲撞预告**：冲撞前身体会膨胀并闪烁
- **碰撞伤害**：15 HP
- **特色**：最基础的地面敌人，引导玩家学习闪避

#### 3. 爆裂晶（Blast Crystal）
- **外观**：不稳定的红色水晶矿团，闪烁发光，体积比矿蝇大
- **AI**：追踪玩家移动，当距离 < 60 px 时停止并开始闪烁（1 秒前摇），然后爆炸
- **爆炸范围**：80 px 圆形 AOE，伤害 20
- **爆炸后**：自身也死亡
- **特色**：群控压力，需要优先击杀或风筝
- **如果被玩家攻击杀死**：不会爆炸，正常死亡

#### 4. 水晶蛛（Crystal Trite）
- **外观**：蜘蛛形态，背部长满尖锐小水晶
- **AI**：跳跃移动（每次跳跃 120 px，间隔 1.5 秒），跳跃轨迹有影子预告
- **落地时**：向 4 个正方向（上下左右）各发射一枚水晶弹幕
- **弹幕速度**：300 px/s，存在 2 秒后消失
- **弹幕伤害**：12 HP
- **特色**：需要观察跳跃影子 + 弹幕方向来闪避

#### 5. 幽魂矿工（Specter Miner）
- **外观**：半透明的矿工亡灵，手持散发幽光的镐子
- **AI**：试图与玩家保持 200-300 px 距离，每 2 秒向玩家发射一枚直线弹幕
- **弹幕**：缓慢但跟踪性强的幽光球（速度 250 px/s，存在 3 秒）
- **受伤时**：短暂闪现到房间随机位置（类似受伤传送）
- **弹幕伤害**：18 HP
- **碰撞伤害**：无（不造成碰撞伤害，纯远程）
- **HP**：3（稍硬）
- **特色**：远程压力怪，需要主动接近击杀

### 3.3 难度分布

| 难度 | 出现的怪物 | 每房间怪物数 |
|------|-----------|-------------|
| Lv1 | 矿蝇、碎石怪 | 3-5 |
| Lv2 | 矿蝇、碎石怪、爆裂晶 | 4-6 |
| Lv3 | 矿蝇、碎石怪、爆裂晶、水晶蛛 | 4-7 |
| Lv4 | 全部 5 种 | 5-8 |
| Lv5 | 全部 5 种 + 精英变体 | 5-10 |

### 3.4 精英变体（Lv5 专属，20% 概率替换普通怪）

| 精英 | 基础怪 | 变化 |
|------|--------|------|
| 矿蝇群 | 矿蝇 | 分裂：死亡后变成 2 只小矿蝇（半血） |
| 大碎石 | 碎石怪 | 尺寸 1.5×，HP×2，冲撞不可打断 |
| 连锁爆裂 | 爆裂晶 | 爆炸后向 8 方向发射慢速弹幕 |
| 毒蛛 | 水晶蛛 | 弹幕命中造成 3 秒减速 |
| 矿工长 | 幽魂矿工 | 弹幕变为 3 连发扇形 |

## 四、Boss 设计（3 个）

### 4.1 Boss 总览

| # | 名称 | 英文 | 出现条件 | HP | 机制关键词 |
|---|------|------|---------|-----|----------|
| 1 | **矿核巨像** | Mine Core Golem | Lv1-2 | 30 | 锤击地面→冲击波，召唤小碎石 |
| 2 | **水晶巢母** | Crystal Broodmother | Lv3-4 | 40 | 产卵孵化小水晶蛛，弹幕旋转阵 |
| 3 | **深渊凝视者** | Abyss Gazer | Lv5 | 50 | 激光扫射，传送，阶段 2 狂暴 |

### 4.2 详细设计

#### Boss 1: 矿核巨像（Mine Core Golem）
- **外观**：巨大的石制人形，全身镶嵌矿石，单眼发光
- **阶段**：单阶段（HP < 30% 时攻击频率 +50%）
- **攻击模式**：
  1. **锤击**（冷却 3 秒）：抬锤蓄力 0.8 秒 → 砸地 → 产生向玩家方向的冲击波（地面扩散圈，速度 200 px/s，范围 300 px）
  2. **召石**（冷却 5 秒，HP < 50% 启用）：Boss 咆哮 → 房间随机位置出现 2 只碎石怪
  3. **冲撞**（冷却 4 秒，HP < 70% 启用）：短暂蓄力后向玩家直线冲撞 350 px
- **弱点**：锤击后 1 秒硬直，是输出窗口
- **战斗节奏**：躲避冲击波 → 输出 → 清理小怪 → 再输出

#### Boss 2: 水晶巢母（Crystal Broodmother）
- **外观**：巨大的蜘蛛形态，腹部是透明水晶囊，内部可见幼虫
- **阶段**：单阶段（HP < 50% 时产卵频率翻倍）
- **攻击模式**：
  1. **产卵**（冷却 6 秒）：Boss 后退 → 在地面放置 1-2 个「水晶卵」（HP 2，30 秒后孵化成水晶蛛，可被玩家提前破坏）
  2. **晶片散射**（冷却 3 秒）：Boss 尖啸 → 向 8 方向发射水晶弹幕（慢速 150 px/s，存在 3 秒）
  3. **突袭**（冷却 4 秒）：Boss 向玩家跳跃，落地产生小型冲击波（范围 100 px）
  4. **虫群冲锋**（HP < 30% 时新增，冷却 8 秒）：所有在场的水晶蛛同时向玩家冲刺
- **弱点**：跳跃落地后 1 秒硬直
- **战斗节奏**：破卵优先级 > 输出 Boss > 躲避弹幕

#### Boss 3: 深渊凝视者（Abyss Gazer）
- **外观**：漂浮的暗紫色能量球体，中央有一只不断开合的巨大眼睛，周围环绕扭曲矿脉
- **阶段**：两阶段
  - **阶段 1**（100%-40% HP）：远程弹幕为主
  - **阶段 2**（40%-0% HP）：狂暴，速度+30%，新攻击加入
- **攻击模式（阶段 1）**：
  1. **凝视**（冷却 2.5 秒）：蓄力 0.6 秒 → 发射一道激光（持续 1.5 秒，缓慢旋转追踪玩家，伤害 8/tick，约 4 ticks/秒）
  2. **暗影球**（冷却 4 秒）：发射 3 枚追踪弹（速度 120 px/s，跟踪 2 秒后直飞），伤害 15
  3. **传送**（冷却 6 秒）：Boss 消失 0.5 秒后出现在房间随机位置
- **攻击模式（阶段 2）**：
  1. 以上所有攻击保留，频率 +30%
  2. **矿脉突刺**（新增，冷却 5 秒）：地面出现 5 条裂纹标记（0.8 秒预告），然后尖刺从地面突起，伤害 20
  3. **分裂眼**（新增，冷却 8 秒）：Boss 分裂出 2 个「小凝视者」（HP 2，只会发射慢速单发暗影球），持续 15 秒后自动消失
- **弱点**：激光扫射结束后 0.8 秒停顿；传送后 0.5 秒硬直
- **战斗节奏**：阶段 1 学习躲避激光 → 阶段 2 场面混乱需要判断优先级

### 4.3 Boss 房间设计

- Boss 房间比普通战斗房间大（800×480）
- **门不锁**：玩家感觉打不过可以逃跑（消耗氧气跑回起始房间直接退出）
- 击败 Boss 后：
  1. 出现 2-3 个高级矿山节点（掉落率比普通战斗房间高一个档次）
  2. 必定掉落 1 个稀有原石（Lv3 geode）
  3. 房间中解锁矿车出口（可以直接返回城镇，或返回前面房间继续采矿）
- Boss 房间的矿山节点不会消失（确保 Boss 战的奖励感）

## 五、玩家攻击系统（已确认）

### 5.1 攻击方式：J 键近战

- **按键**：J 键 → `attack` 输入动作（需要新增到 `project.godot` Input Map）
- **攻击范围**：玩家前方 80 px 扇形（约 120°）
- **攻击伤害**：基础 3 HP（装备可加成）
- **攻击冷却**：0.4 秒（约 2.5 次/秒）
- **攻击动画**：挥镐动作（复用或新制镐的挥动帧）
- **击退**：命中后击退怪物 60 px（小型怪）/ 30 px（中型怪）/ 0（Boss）
- **状态**：攻击时进入 `ATTACK` 状态（0.3 秒），期间不可移动/不可采矿

### 5.2 与现有系统的关系

- J 键攻击不影响 Shift 奔跑、E 键交互（采矿/躲藏）
- 挖掘矿山时（MINING 状态）无法攻击
- 隐藏时（HIDDEN 状态）可以攻击但会破隐
- 死亡时（DEAD 状态）无法攻击

### 5.3 怪物受击反馈

- **闪白**：受击后 0.1 秒白色闪烁
- **击退**：根据怪物体重有不同击退距离
- **血量**：见第三章怪物表中的 HP 值
- **死亡动画**：闪烁后消失（或爆炸，爆裂晶除外——被攻击杀死不爆炸）

## 六、产出机制

### 6.1 怪物掉落（铜板）

怪物死亡时掉落少量铜板（作为小奖励，主要产出仍是矿山节点）：

| 怪物 | 铜板掉落 |
|------|---------|
| 矿蝇 | 1-2 |
| 碎石怪 | 2-3 |
| 爆裂晶 | 2-4 |
| 水晶蛛 | 3-4 |
| 幽魂矿工 | 3-5 |

Boss 掉落：`randi_range(10, 20)` 铜板（仅击败后一次性掉落）。

铜板直接进入 `GameRuntime.wallet`（不需要走热栏）。

### 6.2 矿山节点出现逻辑

战斗房间 → 杀光所有敌人 → 房间中央出现 1-3 个矿山节点（`SmallMine` / `DeepMine`）

```
number_of_mines = randi_range(1, 3)
spawn_positions = _generate_spread_positions(room_center, number_of_mines, min_distance=80)
```

- 矿山节点类型根据难度决定：
  | 难度 | 节点类型 | 分布 |
  |------|---------|------|
  | Lv1 | SmallMine 100% | — |
  | Lv2 | SmallMine 80%, DeepMine 20% | — |
  | Lv3 | SmallMine 60%, DeepMine 40% | — |
  | Lv4 | SmallMine 40%, DeepMine 60% | — |
  | Lv5 | DeepMine 100% | Boss 房间还有星辰矿掉落加成 |

### 6.3 掉落率（保持现有系统）

沿用现有的 `mine_stats.gd` 掉落表，但难度影响：
| 难度 | L1 掉落 | L2 掉落 | L3 掉落 | 幸运 | 星辰矿 |
|------|---------|---------|---------|------|--------|
| Lv1 | 75% | 20% | 5% | 0.30 | 2% |
| Lv2 | 65% | 28% | 7% | 0.25 | 2% |
| Lv3 | 50% | 35% | 15% | 0.20 | 3% |
| Lv4 | 35% | 40% | 25% | 0.15 | 4% |
| Lv5 | 25% | 40% | 35% | 0.15 | 5% |

Boss 房间矿山节点掉落率额外 +1 档（如 Lv3 Boss 房间按 Lv4 掉落率计算）。

### 6.4 完整一次矿洞探险的产出预估

以 Lv3（5 战斗房间 + 1 Boss = 7 总房间）为例：
- 5 战斗房间 × 平均 2 个节点 × 平均 3 段 × Lv3 掉落率 → ~30 个矿石
- 1 Boss 房间 × 2.5 个节点（取 2-3 中值）× 3 段 × Lv4 掉落率 → ~7.5 个矿石
- 总计约 37 个矿石（其中 5-9 个 L3 稀有，1-2 个星辰矿）

### 6.5 氧气和负重系统（保持现有）

- **氧气**：不做改动。起始房间有氧气泵。清理房间本身也消耗氧气，增加紧迫感。
- **负重**：不做改动。挖到的矿石进入热栏，重量影响速度/噪音/氧气消耗。
- **退出**：Boss 房间和起始房间各有一个矿车出口。击败 Boss 后建议直接从 Boss 房间退出（节约氧气），但也可以回头继续采之前没采完的矿山。

## 七、局外成长：难度选择

### 6.1 进入矿洞时选择难度

当前铁匠 NPC 弹出「普通矿洞 / 深层矿洞」面板。重设计后变为：

```
┌──────────────────────────────┐
│     选择矿洞难度             │
│                              │
│  ★ Lv1  浅层矿道  （免费）  │
│  ★★ Lv2  中层矿道  （免费） │
│  ★★★ Lv3  深层矿道 （1门票）│
│  ★★★★ Lv4  深渊裂隙 （2门票）│
│  ★★★★★ Lv5  巨兽之喉 （3门票）│
│                              │
│  [取消]                      │
└──────────────────────────────┘
```

- Lv1-2：免费进入（矿洞是小镇的公共资源）
- Lv3-5：消耗矿洞门票（对应铁匠的「深层入场券」改为通用的「矿洞门票」）
  - Lv3 = 1 张，Lv4 = 2 张，Lv5 = 3 张
- 门票获取：铁匠每天赠送 1 张 + 可购买（价格递增）
- 高难度 = 更好掉落 + 更难怪物 = 风险收益权衡

### 6.2 难度解锁条件

| 难度 | 解锁条件 |
|------|---------|
| Lv1 | 始终可用 |
| Lv2 | 累计完成 3 次 Lv1 矿洞 |
| Lv3 | 累计完成 5 次矿洞（任意难度） |
| Lv4 | 鉴定过 1 块星辰矿 |
| Lv5 | 累计赠予 ≥3 块星辰矿（善人线）或 累计卖出 ≥3 块星辰矿（恶人线） |

## 十、风险与注意事项

1. **现有 test_scene.tscn 的保留**：旧矿洞场景保留而不删除，作为降级方案直到新地牢系统稳定。
2. **氧气系统适配**：房间之间直接门连接（无走廊），穿越门不额外消耗氧气。只在房间内采矿/战斗消耗。
3. **稳定性系统**：保留 StabilitySystem 对敌人检测范围的影响（创造一致的难度感）。
4. **近战攻击状态机**：新增 `ATTACK` 状态到 `player.gd` 状态机。攻击期间（0.3s）不可移动/采矿。
5. **Input Map**：需要在 `project.godot` 新增 `attack` 动作绑定 J 键。
